# 轮询规范

> 本文档定义智能体在长会话中周期性检查知识库更新的行为规则。
> 轮询机制是集体进化闭环的控制杆，决定了知识同步的速度和成本。

---

## 一、轮询判定树

每次收到用户消息时，依次执行以下判定：

```
┌─────────────────────────────────────┐
│ 收到用户消息                         │
└────────────────┬────────────────────┘
                 ▼
      ┌────────────────────┐
      │ 免轮询窗口？         │
      │ 轮次<5 且 时长<5分   │────是──→ 跳过轮询，正常处理
      └────────────────────┘
                 │ 否
                 ▼
      ┌────────────────────┐
      │ 含强制轮询关键词？   │
      │ 新信息/更新/检查/    │────是──→ 强制全量轮询
      │ 最新/其他智能体名称  │
      └────────────────────┘
                 │ 否
                 ▼
      ┌────────────────────┐
      │ 满足定时条件？       │
      │ 距上次>30min        │────是──→ 全量轮询
      │ 或 距上次>10轮       │
      └────────────────────┘
                 │ 否
                 ▼
            跳过轮询
            正常处理
```

**注意：已取消轻量轮询。** 所有轮询均为全量轮询。原因：轻量轮询只读部分文档，缺少关键上下文，曾导致智能体（codex）做出灾难性决策（差点全覆盖其他系统）。

## 二、轮询类型

### 2.1 调度中心读取（全量轮询的第一步）

读调度中心是**最先执行且必须执行**的步骤：

```
GET /api/notes/调度中心/配置.md
  → 对比最后更新日期
  → 有变更 → 读取新参数 → 调整自身行为
  → 无变更 → 跳过

GET /api/notes/调度中心/任务协调.md（可选）
  → 检查是否有分配给自己的任务
  → 有 → 执行任务
```

> **关于公告.md**：公告是第二引导层（固定不可变），在全量连接时读取一次即可。
> 日常轮询不需要重复读取公告，它不属于调度中心。

调度中心是全局智能体的操作中枢。一次写入配置/任务，所有Agent在下一次轮询时自动感知。

### 2.2 全量轮询

- **触发场景**: 定时到达 或 关键词触发
- **检查目标**: 按优先级分组并行读取

| 优先级 | 文档 | 读取成本 |
|--------|------|----------|
| 高 | 周总结.md、月总结.md、系统信息.md | 3次GET |
| 中 | 知识库使用规范.md（比对最后更新行） | 1次GET |
| 中 | 自身更新总结.md | 1次GET |
| 低 | 其他活跃智能体的更新总结（按Key绑定表） | N次GET |

- **输出**: 差异报告，或"无变化"
- **成本**: 典型的5+N次GET

## 三、查重与通知抑制

- 同一文档的同一变化最多提醒用户3次
- 第3次仍被忽略后 → 静默该文档的轮询提醒，直到下次文件更新产生新的hash
- 被静默的文档在轮询记录中标记 `(silent)`

## 四、轮询记录

轮询记录写入智能体自身的更新总结中（不使用独立的连接状态文件）：

```
## 轮询记录
- 最后轮询: YYYY-MM-DD HH:MM（最近一次轮询执行时间）
- 轮询次数: N（累计执行轮询次数）
- 发现更新次数: M（累计发现文档变化的次数）
- 忽略提醒次数: K（用户累计忽略提醒的次数）
- 轮询快照:
  - 周总结: 2026-06-22 (10:00)
  - 月总结: 2026-06-22 (09:00)
  - 自身更新总结: 2026-06-22 (10:30)
  - 系统信息: 2026-06-22 (10:00)
```

快照中的日期时间为文档的"最后更新"字段。比对时仅需检查该行内容是否一致。

## 五、注意事项

- 轮询判定在用户消息处理的开头执行，不影响主任务
- 轮询发现更新后先通知用户，等用户确认后再加载，不自动注入上下文
- 轮询记录每次轮询后立即写入知识库，确保持久化
